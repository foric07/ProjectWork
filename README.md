# CRM-система для малого бизнеса (MVP)

## Цель
Создать базу данных для управления взаимодействием с клиентами: учёт клиентов, контактов, сделок, истории взаимодействий и аналитическая отчётность.

## Основные возможности
- Хранение структурированных данных о клиентах, сделках, сотрудниках, продуктах, активностях.
- Хранение неструктурированных данных в JSONB (контакты, сделки) и XML (логи импорта).
- Аналитические отчёты: воронка продаж, эффективность сотрудников, сегментация клиентов.
- Оконные функции для глубокого анализа: LAG, RANK, SUM() OVER и др.
- Представления для упрощения доступа к агрегированным данным.
- Хранимые процедуры для бизнес-операций (перемещение сделок по этапам).
- Оптимизация производительности: индексы, материализованные представления, партиционирование (по желанию).

## Схема данных (основные таблицы)
1. employees – сотрудники
2. lead_sources – источники лидов
3. companies – компании-клиенты
4. contacts – контактные лица
5. pipeline_stages – этапы воронки продаж
6. products – продукты/услуги
7. deals – сделки
8. deal_items – позиции сделок
9. activities – взаимодействия
10. activity_types – типы активностей
11. xml_import_log – пример XML данных

Все таблицы связаны внешними ключами, что обеспечивает целостность и удобство аналитики.

## Как запустить
1. Установите PostgreSQL 13+.
2. Выполните `schema.sql` для создания структуры БД.
3. Выполните `seed.sql` для наполнения тестовыми данными.
4. Далее можно выполнять запросы из `analytical_queries.sql`, создавать представления (`views.sql`), процедуры (`procedures_functions.sql`) и отчёты (`reports.sql`).

## Используемые технологии
- SQL (PostgreSQL)
- JSONB, XML
- Оконные функции (LAG, RANK, SUM OVER)
- Представления, хранимые процедуры, индексы, EXPLAIN
